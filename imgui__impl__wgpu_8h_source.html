1// dear imgui: Renderer for WebGPU
2// This needs to be used along with a Platform Binding (e.g. GLFW, SDL2, SDL3)
3// (Please note that WebGPU is a recent API, may not be supported by all browser, and its ecosystem is generally a mess)
4
5// When targeting native platforms:
6// - One of IMGUI_IMPL_WEBGPU_BACKEND_DAWN or IMGUI_IMPL_WEBGPU_BACKEND_WGPU *must* be provided.
7// When targeting Emscripten:
8// - We now defaults to IMGUI_IMPL_WEBGPU_BACKEND_DAWN is Emscripten version is 4.0.10+, which correspond to using Emscripten '--use-port=emdawnwebgpu'.
9// - We can still define IMGUI_IMPL_WEBGPU_BACKEND_WGPU to use Emscripten '-s USE_WEBGPU=1' which is marked as obsolete by Emscripten.
10// Add #define to your imconfig.h file, or as a compilation flag in your build system.
11// This requirement may be removed once WebGPU stabilizes and backends converge on a unified interface.
12//#define IMGUI_IMPL_WEBGPU_BACKEND_DAWN
13//#define IMGUI_IMPL_WEBGPU_BACKEND_WGPU
14
15// Implemented features:
16// [X] Renderer: User texture binding. Use 'WGPUTextureView' as ImTextureID. Read the FAQ about ImTextureID/ImTextureRef!
17// [X] Renderer: Large meshes support (64k+ vertices) even with 16-bit indices (ImGuiBackendFlags_RendererHasVtxOffset).
18// [X] Renderer: Expose selected render state for draw callbacks to use. Access in '(ImGui_ImplXXXX_RenderState*)GetPlatformIO().Renderer_RenderState'.
19// [X] Renderer: Texture updates support for dynamic font system (ImGuiBackendFlags_RendererHasTextures).
20// Missing features or Issues:
21// [ ] Renderer: Multi-viewport support (multiple windows), useful for desktop.
22
23// You can use unmodified imgui_impl_* files in your project. See examples/ folder for examples of using this.
24// Prefer including the entire imgui/ repository into your project (either as a copy or as a submodule), and only build the backends you need.
25// Learn about Dear ImGui:
26// - FAQ https://dearimgui.com/faq
27// - Getting Started https://dearimgui.com/getting-started
28// - Documentation https://dearimgui.com/docs (same as your local docs/ folder).
29// - Introduction, links and more at the top of imgui.cpp
